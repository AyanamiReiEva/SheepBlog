## 第二阶段：博客核心功能开发（依赖第一阶段全部完成）
### 前端风格核心规范（强制对齐 https://magazine.sebastianraschka.com/ 站点风格）
采用极简专业的技术博客设计理念，**100%以内容为核心，完全弱化装饰性元素，极致优化长文阅读体验**，具体规范如下：
1. **配色系统**
   - 主背景：纯白色（#FFFFFF）/ 极深灰（#121212，暗色模式），无渐变、无背景纹理
   - 文字：主文本使用深灰（#1a1a1a）/ 浅灰（#e0e0e0，暗色模式），对比度符合WCAG AAA级标准
   - 强调色：仅使用1种低饱和度主色（深蓝/深绿系）用于链接、高亮和交互元素，无多余彩色元素
   - 分割线/边框：使用极浅灰（#f0f0f0）/ 深灰（#2a2a2a，暗色模式），线条粗细不超过1px
2. **排版系统**
   - 字体层级：标题使用优雅衬线字体（如Georgia、Inter Serif），正文使用高可读性无衬线字体（如Inter、系统无衬线字体）
   - 行高：正文行高1.6-1.75，标题行高1.2-1.3，代码块行高1.5
   - 字号：正文字号16-18px（桌面端），15-16px（移动端），标题层级清晰区分
   - 字重：正文使用400常规字重，标题使用600-700字重，避免过粗或过细的字体
   - 间距：段落间距1.5-2倍行高，内容区左右留白充足，避免内容贴边
3. **布局规范**
   - 整体布局：单栏/窄双栏布局，内容区域最大宽度限制在700-800px（长文阅读最优宽度）
   - 导航：极简顶部/侧边导航，仅保留必要的链接，无复杂下拉菜单
   - 文章列表：无卡片设计、无阴影、无圆角，仅用标题+摘要+发布时间的纯文本列表形式
   - 文章详情页：无侧边栏干扰，仅保留目录导航（可选固定右侧），内容居中展示
4. **交互与体验**
   - 悬停效果：极简，仅文字颜色轻微变化或下划线，无放大、阴影等夸张效果
   - 动画：所有交互动画时长不超过200ms，过渡平缓自然，无花哨动效
   - 阅读体验：支持暗色/亮色模式自动切换，无广告、无弹窗、无冗余元素干扰阅读
   - 响应式：移动端完全适配，触控区域足够大，排版自动调整到最优阅读尺寸

**所有UI设计必须严格遵循以上规范，减少不必要的视觉干扰，提供沉浸式长文阅读体验。**

### 3.0 基础能力层开发（需优先完成，无内部依赖）

- 后端开发智能体提示词：负责根据需求开发Next.js API Routes/tRPC接口
- 架构师智能体提示词：负责评审代码的架构合理性、性能、安全性、可维护性
- 测试工程师智能体提示词：负责根据需求和业务代码生成测试用例
