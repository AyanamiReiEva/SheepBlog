# Pull Request规范

## PR标题格式

```
[<type>] <short description>
```

## Type类型

- FEATURE: 新功能
- FIX: 修复bug
- DOCS: 文档
- REFACTOR: 重构
- TEST: 测试
- CHORE: 其他

## PR描述模板

```
## 变更描述
简要描述此次变更的内容和目的。

## 关联需求
- 需求ID: xxx
- 需求文档: [链接]

## 变更类型
- [ ] 新功能
- [ ] Bug修复
- [ ] 文档更新
- [ ] 代码重构
- [ ] 性能优化

## 测试覆盖
- [ ] 单元测试
- [ ] 集成测试
- [ ] E2E测试

## 检查清单
- [ ] 代码通过ESLint检查
- [ ] 代码已格式化
- [ ] 所有测试通过
- [ ] 已更新相关文档
- [ ] 已添加必要的注释
```
